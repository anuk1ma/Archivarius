# КУРСОВОЙ ПРОЕКТ

## ВВЕДЕНИЕ

В условиях активного развития цифровых технологий пользователи все чаще взаимодействуют с книгами, каталогами и сервисами заказа через web-интерфейсы. Современный пользователь ожидает, что поиск нужного издания, просмотр информации о книге, формирование заказа и управление личным кабинетом будут доступны в одном удобном приложении. Особенно это актуально для тематических книжных сервисов, ориентированных на структурированный каталог и быстрый доступ к литературе.

Актуальность темы курсового проекта обусловлена необходимостью разработки современного web-приложения, которое объединяет функции поиска книг, работы с избранным, корзиной, заказами и административным управлением. Такие системы востребованы как в электронной коммерции, так и в цифровых библиотечных сервисах, поскольку позволяют автоматизировать основные пользовательские сценарии и сделать работу с каталогом книг удобной и наглядной.

Целью курсового проекта является разработка системы поиска и заказа книг в библиотеке в формате full-stack web-приложения `Archivarius`, ориентированного на каталог классической литературы.

В рамках проекта решаются задачи проектирования структуры системы, выбора инструментов разработки, создания клиентской и серверной частей, подключения базы данных PostgreSQL, реализации пользовательских и административных функций, а также проведения функционального тестирования.

Практическая значимость проекта состоит в том, что созданное приложение может быть использовано как демонстрационный прототип книжного web-сервиса с поддержкой авторизации, поиска, заказа и управления каталогом.

## 1. ЦЕЛЬ И ЗАДАЧИ ПРОЕКТА

Разработка курсового проекта направлена на закрепление знаний по дисциплине `ПМ04 - Проектирование и обеспечение бесперебойной работы web-сайта`, а также на получение практических навыков проектирования и реализации современного web-приложения.

### 1.1 Цель проекта

Основной целью проекта является создание web-приложения `Archivarius`, предназначенного для поиска, просмотра и заказа книг через удобный пользовательский интерфейс. Разрабатываемая система должна обеспечивать доступ к каталогу мировой классической литературы, поддерживать регистрацию пользователей, оформление заказа, хранение истории заказов и наличие административной панели.

Проект ориентирован на реализацию не только визуальной части интерфейса, но и полноценной серверной логики с базой данных, REST API и разграничением ролей пользователей.

## 2. ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ ПРИЛОЖЕНИЯ

Web-приложение `Archivarius` представляет собой книжный сервис, в котором пользователь может работать с каталогом классической литературы, находить книги по различным параметрам и оформлять условные заказы.

К основным функциональным возможностям приложения относятся регистрация и вход по электронной почте и паролю. После авторизации пользователь получает доступ к личному кабинету, избранному, корзине и истории заказов. Для удобства взаимодействия с системой предусмотрена возможность загрузки аватара по URL и редактирования персональных данных.

Следующим важным блоком является каталог книг. Пользователь может искать книги по названию, автору и жанру, сортировать каталог по популярности и цене, а также открывать подробную карточку каждой книги. В карточке отображаются обложка, название, автор, жанр, издательство, язык, рейтинг, цена и развернутое описание произведения.

Система поддерживает работу с избранным и корзиной. Книга может быть добавлена в список избранных, чтобы пользователь мог быстро вернуться к ней позже. Корзина позволяет собрать несколько книг и затем оформить заказ через единую форму.

Оформление заказа реализовано через отдельную форму, в которой указываются город, адрес и способ оплаты. Пользователь может выбрать оплату при получении либо демонстрационный вариант оплаты картой. При выборе оплаты картой отображаются поля для номера карты, имени владельца, срока действия и CVV. Несмотря на учебный характер оплаты, форма проходит валидацию и создает полноценный сценарий checkout.

Отдельно реализована административная панель. Администратор может добавлять, изменять и удалять книги, просматривать пользователей, удалять обычных пользователей, изменять роли и управлять статусами заказов. Благодаря этому приложение покрывает не только клиентские, но и административные задачи сопровождения каталога.

Дополнительным преимуществом является поддержка двух языков интерфейса: русского и английского. При переключении языка изменяются элементы навигации, кнопки, подписи и локализованные данные по книгам.

## 3. ИСПОЛЬЗОВАННЫЕ СРЕДСТВА И ТЕХНОЛОГИИ

При разработке проекта был выбран стек технологий, позволяющий создать полноценное full-stack web-приложение с современным пользовательским интерфейсом, серверной логикой и реляционной базой данных.

Для клиентской части использовались `React` и `Vite`. React обеспечивает компонентный подход к построению интерфейса и удобную организацию кода, а Vite позволяет быстро запускать проект в режиме разработки и эффективно собирать production-версию приложения.

Для навигации между страницами использовалась библиотека `react-router-dom`, что позволило реализовать SPA-маршрутизацию и разделить публичные, пользовательские и административные маршруты.

Серверная часть была реализована на `Node.js` с использованием `Express`. Такое решение позволило организовать REST API, работу middleware, проверку прав доступа, обработку ошибок и взаимодействие с PostgreSQL.

В качестве системы управления базой данных был выбран `PostgreSQL`, размещенный в облачном сервисе `Neon`. Это обеспечило надежное хранение данных о пользователях, книгах, корзине, избранном и заказах. Для деплоя были использованы платформы `Vercel` для frontend-части и `Render` для backend-части.

Для авторизации применяются `JWT`-токены, а пароли пользователей сохраняются в виде хешей с использованием `bcrypt`. Данный подход обеспечивает базовый уровень безопасности для учебного web-приложения.

Для подготовки документации и диаграмм были использованы Markdown, Word-форматирование и графические схемы архитектуры, навигации и базы данных.

## 4. ОПИСАНИЕ ИНТЕРФЕЙСОВ

Интерфейс приложения выполнен в стилистике старой библиотеки и ориентирован на каталог мировой классической литературы. В дизайне используются теплые кофейные оттенки, крупные карточки книг, декоративные акценты и атмосферные иллюстрации, создающие визуально цельный образ сервиса.

На главной странице пользователь знакомится с концепцией сервиса, после чего переходит в каталог. Каталог содержит поисковую строку, фильтры по жанру и сортировку по популярности или цене. Карточки книг адаптированы под десктоп, планшет и мобильные устройства.

Карточка книги отображает основную библиографическую информацию, цену и подробное описание. Из карточки пользователь может добавить книгу в избранное или корзину. В правой верхней части интерфейса размещены язык, корзина и блок аккаунта.

Профиль пользователя включает вкладки с персональными данными, избранными книгами и историей заказов. Корзина оформлена как выдвижная панель, внутри которой находится форма заказа. На мобильных устройствах реализовано выдвигающееся боковое меню, что улучшает удобство навигации.

Административная панель включает формы добавления и редактирования книг, таблицу пользователей и список заказов. Интерфейс админки оформлен в общей стилистике приложения, но сохраняет более практичную структуру для управления данными.

## 5. АРХИТЕКТУРНАЯ МОДЕЛЬ СИСТЕМЫ

Приложение `Archivarius` построено по клиент-серверной архитектуре. Пользователь взаимодействует с frontend-частью через браузер. Клиентская часть отправляет HTTP-запросы к REST API, сервер обрабатывает бизнес-логику и взаимодействует с базой данных PostgreSQL.

Клиентская часть отвечает за отображение интерфейса, маршрутизацию, формы, локализацию и пользовательский опыт. Серверная часть реализует регистрацию, вход, защиту маршрутов, операции с книгами, корзиной, заказами и админскими сущностями. PostgreSQL обеспечивает хранение всех данных системы.

В рамках проектирования были выделены следующие основные сущности: `users`, `books`, `favorites`, `cart_items`, `orders`, `order_items`. Между ними существуют связи, необходимые для поддержки пользовательских сценариев и административных операций.

Для системы были подготовлены диаграммы, отражающие структуру приложения, архитектуру взаимодействия компонентов, ER-модель базы данных, use case пользователей и схему оформления заказа.

## ЗАКЛЮЧЕНИЕ

В ходе выполнения курсового проекта была разработана система поиска и заказа книг в библиотеке в формате современного full-stack web-приложения `Archivarius`. В рамках проекта были изучены особенности предметной области, подобран стек технологий, спроектирована архитектура системы и реализована база данных PostgreSQL.

В результате был создан программный продукт, который включает каталог книг, поиск, фильтрацию, авторизацию пользователей, избранное, корзину, оформление заказа, профиль пользователя и административную панель. Проект также поддерживает двуязычный интерфейс и адаптивную верстку для разных типов устройств.

Поставленная цель была достигнута, а задачи, определенные в начале работы, выполнены в полном объеме. Разработанное приложение может рассматриваться как демонстрационный прототип книжного web-сервиса и служить основой для дальнейшего расширения функционала.

Практическая ценность проекта заключается в том, что он показывает применение современных средств web-разработки в рамках реальной задачи и демонстрирует навыки проектирования, программирования, интеграции базы данных, тестирования и деплоя.
